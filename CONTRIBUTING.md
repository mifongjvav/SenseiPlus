# 贡献指南

## 报告漏洞

bug 是指SenseiPlus出现的错误，适用的错误报告示例如下：

- 输入内容与执行内容不符。
- 当我执行某些功能时，直接崩溃。

为了跟踪错误报告，我们主要使用 **GitHub issues**。在创建issue时，请先：

- 搜索有没有重复的issue。
- 确定这个问题是不是python问题。

## 提交拉取请求

以下是准备代码变更时需要注意的简要清单：

- 确保你熟悉面向对象编程的规则、Python的语法以及你的开发环境。
- 确保你熟悉 [git](https://git-scm.com/) 和 [拉取请求的工作流程 ](https://help.github.com/en/github/collaborating-with-issues-and-pull-requests/proposing-changes-to-your-work-with-pull-requests)。
- **请不要通过 GitHub 网页界面进行代码更改**。

完成修改后，如果要提交到上游的 `dev` 分支，你应该：

- 请提交到 `dev` 分支而不是 `main` 分支
